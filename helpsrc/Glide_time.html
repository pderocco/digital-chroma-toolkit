GLIDE time

These control how long it takes for a voice's pitch to change from one note to another, when {_glide} is being used.

Glide normally only functions in monophonic {Common_keyboard_algorithm} and the poly ordered and poly buffered algorithms. However, it can also be enabled on the normal poly mode via the left {Common_footswitch_mode}. When the footswitch is pressed, voice assignment temporarily becomes monophonic, and glides from note to note.

Glide A always controls {_oscillator} A, and glide B always controls oscillator B, so to use glide one normally {_gang gangs} the {Glide_cluster glide clusters} together.

Toolkit

These are contained in the {Glide_cluster Glide clusters} within the {Glide_section}.

Chroma panel

These are contained in the GLIDE section of the right panel, under the following parameter number:

{sw-6.png height=48}

These appear in the display as Glide Time.

The switch graphics call this a "rate", but since larger values produce longer times, this is really a "time", and this terminology is corrected in the Toolkit and the display.

MIDI

These are accessed via the following {_NRPN NRPNs}:

A 1,0
B 2,0

Values

The values are displayed as times, in milliseconds (ms) or seconds (s). The constant time and exponential shapes use the values in the second column, while the constant slope and glissando shapes use the values in the third column:
